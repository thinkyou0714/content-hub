# ベストプラクティス100(索引)

AI副業・個人開発の発信運用に関するベストプラクティス100個。10カテゴリ x 10個で構成する。
各項目は「要点・根拠(調査ノート出典)・実装(このリポジトリのどこで実現されているか)・関連」の4点で記述され、単なる心得集ではなく**リポジトリの資産(プロンプト・テンプレ・チェックリスト・ワークフロー)に紐付いた実装済みの運用ルール**として機能する。

## 使い方

- 生成・運用で迷ったら、該当カテゴリを開いて判断基準として使う
- 月次レビュー([../checklists/monthly-review.md](../checklists/monthly-review.md))で「現実と合わなくなった項目」を更新する。100個は固定資産ではない
- 根拠の元データは [../research/notes/](../research/notes/) の調査ノート(2026-07時点)。鮮度切れに注意

## 特に重要な10個(迷ったらここから)

運用の生命線になる項目を各カテゴリから1つずつ挙げる。

1. [BP-002](01-strategy.md): 実物主義 — 動いているものしか出さない
2. [BP-018](02-zenn-writing.md): Zennの「人が主体」原則を公開フローで担保する
3. [BP-022](03-x-operations.md): リプライが発生する設計にする
4. [BP-031](04-psychology.md): 冒頭1〜2行を記事タイトルの感覚で作り込む
5. [BP-044](05-ai-generation.md): 人間レビューなしで公開しない
6. [BP-052](06-repurposing.md): 展開は「翻訳」であって「コピペ」ではない
7. [BP-061](07-engagement.md): 自分の投稿へのリプライには24時間以内に人間が返す
8. [BP-071](08-monetization.md): 無料で価値を実証してから有料化する
9. [BP-086](09-indie-dev.md): 失敗談を一級のコンテンツとして扱う
10. [BP-091](10-operations-risk.md): 収益表示には期間・コスト・前提を併記する

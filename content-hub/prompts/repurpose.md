# 生成プロンプト: リパーパス(既存コンテンツの多チャネル展開)

1つのコンテンツを使い切るための展開プロンプト。展開スケジュールは [../workflows/content-pipeline.md](../workflows/content-pipeline.md) の「展開の基本セット」に従う。

## 必読コンテキスト(AIに一緒に渡す)

- 元コンテンツ(Zenn記事など)
- [../voice/style-guide.md](../voice/style-guide.md)
- [../templates/x-post-patterns.md](../templates/x-post-patterns.md)

## 入力変数

```text
元コンテンツ: (記事パス or URL)
展開先: 告知ポスト / 図解ポスト / スレッド / 裏話ポスト / note深掘り案 のいずれか(複数可)
公開からの経過: (当日 / 2〜3日後 / 1週間後 / 2週間後)
反応データ: (どの節・どの一文が反応を得たか。あれば)
```

## プロンプト本文

```text
あなたはTHINK YOU LABのコンテンツ展開アシスタントです。
元コンテンツを指定された展開先向けに再構成してください。

大原則: 「翻訳」であって「コピペ」ではない。各チャネルの文脈・読者・トーンに合わせて作り直す。

展開先別の要件:
【告知ポスト(公開当日)】
- 記事の「一番おいしい一文」を軸に1投稿。リンクは本文に入れずリプ欄用に分離
【図解ポスト(2〜3日後)】
- 記事の核心を1枚図解の構成案(Mermaid or 箇条書きレイアウト)+ 添付文で出力
【スレッド(1週間後)】
- x-thread.md のプロンプトを適用(このファイルからは呼び出し指示だけ出す)
【裏話ポスト(2週間後)】
- 記事に書かなかった失敗・迷い・ボツ案を軸に1投稿。ここが一番人間味を出す場所。
  素材が元コンテンツにない場合は【人間: 裏話素材を記入】を残す
【note深掘り案】
- 有料化する場合の目次案と「無料版との差分」を提案のみ(執筆は note-paid-article.md で)

生成後のセルフチェック:
- [ ] 元コンテンツにない事実・数字を足していないか
- [ ] チャネルの文脈に翻訳されているか(コピペ移植になっていないか)
- [ ] 展開の時期に合った内容か(公開直後に裏話を出さない等)
```

## 運用メモ

- 1記事から最低5展開が標準ノルマ(告知・図解・スレッド・裏話+計測後の判断1つ)
- 展開の反応もネタ帳([../research/idea-backlog.md](../research/idea-backlog.md))に記録し、次のネタ選定に使う
- 反応が良かった展開は3ヶ月後に「切り口を変えて再展開」してよい(同一文面の再投稿は不可)

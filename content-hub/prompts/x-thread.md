# 生成プロンプト: Xスレッド(ツリー投稿)

## 必読コンテキスト(AIに一緒に渡す)

- [../voice/style-guide.md](../voice/style-guide.md)
- [../templates/x-post-patterns.md](../templates/x-post-patterns.md)
- 元ネタ(Zenn記事本文、またはネタ帳の行+人間の素材)

## 入力変数

```text
元ネタ: (記事パス or 体験・ノウハウの素材)
柱: PILLAR-X
ペルソナ: P1 / P2 / P3
本数: 5〜8本(既定は6本)
狙い: ノウハウの出し切り / 記事への導線 / ストーリー共有
```

## プロンプト本文

```text
あなたはTHINK YOU LABのXスレッド作成アシスタントです。
元ネタをXスレッド(ツリー投稿)に再構成してください。

制約:
1. 1本目 = 結論 or 強い問い + 「スレッドで解説する」の予告。1本目だけで独立して拡散に耐える品質にする
2. 2本目以降は「1ポスト1ポイント」。各ポストが単体で引用・保存される前提で書く
3. 記事の丸写しではなく、X向けの口語に翻訳する(文脈翻訳。コピペ移植は禁止)
4. 途中に1箇所、読者の「あるある」または失敗談を入れて感情の起伏を作る
5. 最終ポスト = まとめ + 次の行動(記事リンクはこの最終ポストのリプ欄に置く設計にする)
6. 実数・体験は元ネタにあるものだけ。創作禁止
7. 各ポストは全角140字以内を目安(長文Premium前提にしない)

出力形式:
- ポストごとに「--- n本目 ---」で区切る
- 最後に「リプ欄用リンクポスト」を別途出力

生成後のセルフチェック:
- [ ] 1本目単体で拡散に耐えるか
- [ ] 各ポストが単体で意味が通るか
- [ ] 元ネタにない数字・エピソードを足していないか
```

## 生成後の流れ

1. 人間レビュー(通し読みで「自分の声か」を確認)
2. [../checklists/x-pre-post.md](../checklists/x-pre-post.md) を適用
3. 投稿タイミングは記事公開の約1週間後が基本([../workflows/content-pipeline.md](../workflows/content-pipeline.md) の展開セット)

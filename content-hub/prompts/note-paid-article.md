# 生成プロンプト: 有料note原稿

## 必読コンテキスト(AIに一緒に渡す)

- [../voice/style-guide.md](../voice/style-guide.md)(noteのトーン調整の節)
- [../templates/note-article-template.md](../templates/note-article-template.md)(無料部分/有料部分の構造)
- [../strategy/channels.md](../strategy/channels.md)(noteの運用ルール)
- 需要の根拠(Zenn/Xでの反応データ)

## 入力変数

```text
テーマ: (Zenn/Xで反応が実証されたものだけ。実証データを添える)
ペルソナ: P1 / P3
人間が提供する素材: (手順・テンプレ・設定ファイル・数字・失敗談)
価格帯の想定: (人間が決める。AIは提案のみ可)
無料で既に公開している範囲: (Zenn記事URL等)
```

## プロンプト本文

```text
あなたはTHINK YOU LABの有料note執筆アシスタントです。
テンプレートに従い、有料noteの原稿を作成してください。

制約:
1. 無料部分だけで「読者が1つ持ち帰れる」価値を出す(無料部分=信頼の証明)
2. 有料部分は「深掘り・完全な手順・テンプレ・時短」の対価にする。
   「ここでしか知れない稼ぎ方」のような情報商材型の売り方は禁止
3. 収益・実績に触れる場合は、期間・かかったコスト・前提条件を必ず併記する(景表法・ステマ規制対応)
4. 「誰でも」「必ず」「簡単に稼げる」等の表現は禁止
5. 無料公開済みの内容(Zenn記事)との差分を明確にし、二重売りにしない
6. 体験・数字は提供素材にあるものだけ。不足は【人間: ○○を記入】で残す
7. 文体は「俺」の熱量を保ちつつ、購入者への丁寧さを1段上げる

生成後のセルフチェック:
- [ ] 無料部分単体で価値があるか
- [ ] 有料部分の価値の根拠(何が時短・完全になるか)を説明できるか
- [ ] 収益表示に期間・コスト・前提が併記されているか
- [ ] 誇大表現ゼロか
```

## 生成後の流れ

1. 人間がプレースホルダーを埋め、価格を決定([../workflows/human-tasks.md](../workflows/human-tasks.md) HT-6)
2. 法務系チェック([../best-practices/10-operations-risk.md](../best-practices/10-operations-risk.md) 参照)
3. 公開後、Xで無料部分の価値を軸に告知([repurpose.md](repurpose.md))

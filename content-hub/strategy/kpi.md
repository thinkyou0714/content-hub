# KPI・計測設計

数字は「伸ばすため」ではなく「次の一手を決めるため」に見る。
計測は月次レビュー([checklists/monthly-review.md](../checklists/monthly-review.md))で行い、結果をこのファイルの下部に追記していく。

## 北極星指標(North Star)

**「発信経由で行動した読者の数」** — 記事の手順を試した報告、質問、フォロー転換、有料購読。
インプレッションではなく「読者の行動」を最上位に置く。

## チャネル別KPI

| チャネル | 主KPI | 副KPI | 見ない指標 |
|---|---|---|---|
| X | プロフィールアクセス数、フォロワー純増 | 返信数、ブックマーク数 | いいね総数(質と相関しない) |
| Zenn | 記事ごとのいいね・ブックマーク | トレンド入り有無、被リンク | PV単体(流入源とセットでのみ見る) |
| note | 購入数・購入率 | スキ数、フォロワー | 無料部分のPV単体 |

## フェーズ別の目安(固定目標ではなく判断の基準線)

| フェーズ | 状態 | 注力 |
|---|---|---|
| Phase 0(現在) | Zenn記事4本、シリーズ1本完結 | Xの毎日運用の習慣化と型の検証 |
| Phase 1 | X運用が週次ルーチンで安定 | 反応データをネタ選定に反映するループ確立 |
| Phase 2 | 反応の良い型が3つ以上見つかる | note有料の初出し、Zenn本の検討 |
| Phase 3 | 発信経由の収益が発生 | 収益源の分散と「実録」コンテンツ化(PILLAR-4) |

## 計測の実務

- 週次: X アナリティクスの数字を [workflows/weekly-routine.md](../workflows/weekly-routine.md) の振り返り欄に記録(人間5分)
- 月次: 月次レビューで「伸びた型/滑った型」を特定し、[templates/x-post-patterns.md](../templates/x-post-patterns.md) と [best-practices/](../best-practices/) を更新
- 数字の解釈・改善案の立案はAIに任せてよいが、**目標設定と撤退判断は人間が行う**

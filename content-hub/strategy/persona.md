# ターゲット読者ペルソナ

すべての文章は、下の3人のうち**誰に向けて書くか**を決めてから生成する。
「全員向け」の文章は誰にも刺さらない。プロンプトに必ずペルソナ番号を渡すこと。

## P1: 自動化に憧れる会社員(メインターゲット)

| 項目 | 内容 |
|---|---|
| 属性 | 25〜40歳の会社員。ITリテラシーは中程度(コピペでコードは動かせる) |
| 状況 | 本業だけの収入に不安。副業を探しては挫折。AIツールの話題は追っている |
| 痛み | 「時間がない」「何から始めればいいかわからない」「続かない」 |
| 欲求 | 会社に依存しない収入の柱。自分の手で何かが動く手応え |
| 刺さる表現 | Before/Afterの実数、「毎朝Slackに届く」のような生活描写、初期費用の安さ |
| 避ける表現 | 専門用語の羅列、前提知識の暗黙要求、「簡単に稼げる」系の煽り(不信感を持たれる) |

## P2: 手を動かすエンジニア・個人開発者

| 項目 | 内容 |
|---|---|
| 属性 | 20〜40代のエンジニア(本業/趣味問わず)。Zennの主要読者層 |
| 状況 | 技術は追えているが、発信や個人開発を収益に繋げられていない |
| 痛み | 「作ったのに誰にも使われない」「発信が続かない」「マネタイズが下手」 |
| 欲求 | 技術力を資産に変える具体的な方法。リアルな数字とアーキテクチャ |
| 刺さる表現 | 構成図、コード、落とし穴(トラブルシュート)、コスト内訳、設計判断の理由 |
| 避ける表現 | 技術的にあいまいな説明、ポエムだけで実装がない記事 |

## P3: AI活用で先行したい発信者・スモールビジネス層

| 項目 | 内容 |
|---|---|
| 属性 | すでに発信や小さい事業をしていて、AIで効率化・差別化したい層 |
| 状況 | ChatGPTは使っているが「仕組み化」まではできていない |
| 痛み | 「毎回手作業でAIに指示している」「品質が安定しない」「時間が溶ける」 |
| 欲求 | 再現可能なワークフロー。自分の業務に移植できる設計図 |
| 刺さる表現 | パイプライン全体像、人間とAIの役割分担、運用コストと時間の実数 |
| 避ける表現 | ツール紹介だけで運用の実態がない内容 |

## チャネルとペルソナの対応(基本則)

- **Zenn** → P2 が主、P1 が従(技術的正確さを最優先)
- **X** → P1 が主、P2/P3 が従(共感と実数のフックを最優先)
- **note(有料)** → P1/P3(再現手順と時短の対価として課金してもらう)

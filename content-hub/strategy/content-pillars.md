# コンテンツの柱(Content Pillars)

発信するテーマを5本の柱に固定する。ネタ選定・生成時は必ずどの柱かをタグ付けする。
柱から外れるネタは「実験枠」(全体の1割まで)として明示的に扱う。

## 柱の一覧

| # | 柱 | 内容 | 主ペルソナ | 主チャネル |
|---|---|---|---|---|
| PILLAR-1 | 副業自動化システム | n8n x Claude Code x Obsidian の構築・運用記録。WF設計、hooks、パイプライン | P1/P2 | Zenn → X |
| PILLAR-2 | AI文章・画像生成の実務 | 生成AIをコンテンツ制作に組み込む方法。プロンプト設計、品質担保、コスト | P3/P2 | Zenn / note |
| PILLAR-3 | 個人開発の記録 | 作ったものの開発ログ、技術選定、公開までの道のり、使われるまでの試行錯誤 | P2 | X → Zenn |
| PILLAR-4 | 副業のリアル(数字と時間) | 収益・コスト・作業時間の実数公開。うまくいかなかったことも含む | P1 | X / note |
| PILLAR-5 | AIとの働き方 | 人間とAIの役割分担、レビューの仕方、AIに任せて失敗した話 | P1/P3 | X / Zenn(idea) |

## 柱ごとの生成比率(週次の目安)

- X投稿: PILLAR-1/3/4 を中心に週の投稿の7割。残りをPILLAR-2/5と実験枠
- Zenn記事: PILLAR-1/2/3 から月2〜4本(tech型)。PILLAR-5はidea型で月0〜1本
- note: PILLAR-2/4 の深掘りを月0〜1本(Zenn/Xで反応が実証されたテーマのみ有料化する)

## ネタの採用基準(3問テスト)

新しいネタは以下の3問すべてにYESで採用。1つでもNOなら保留箱へ。

1. **実物があるか** — 自分が動かした/作ったものについて語れるか
2. **柱に乗るか** — 5本の柱のどれかに属するか(または明示的な実験枠か)
3. **読者の行動が変わるか** — 読んだ人が「真似できる・試せる・判断が変わる」か

## シリーズ戦略

単発記事より「シリーズ」を優先する(既刊: 「AIと自動化で副業システムを作る」全4回)。

- シリーズは相互リンクで回遊を生み、読者の信頼を累積させる
- 新シリーズを起こすときは、先にXで各回のテーマを小出しにして反応を検証してから書く
- シリーズ完結後はまとめ記事(ハブ記事)を作り、X固定ポストとプロフィールから導線を張る

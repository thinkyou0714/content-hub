# チェックリスト: Zenn公開前

記事公開の最終ゲート。機械チェック(AI実行)→ 人間チェックの2段構え。
1つでもNGなら公開を延期する(`publish_scheduled` をずらすだけ。焦らない)。

## 機械チェック(AIに実行させる)

- [ ] `npm run lint` が全部通る(markdownlint + textlint + zenn frontmatter検証)
- [ ] frontmatter: `title` / `emoji` / `type` / `topics`(5個以内) / `publish_scheduled`(09:00 JST) が正しい
- [ ] ドラフト用フィールド(`hold_reason` / `next_action` / `stage`)が削除されている
- [ ] `【人間: 】` プレースホルダーが残っていない
- [ ] リンク切れがない(シリーズ相互リンク含む)
- [ ] 文体ガイド([../voice/style-guide.md](../voice/style-guide.md))の禁止事項・AIっぽさが残っていない

## 人間チェック(30分)

### 事実(最重要)

- [ ] すべての数字・スクショ・構成図が実物か(1つでも盛りがあれば公開しない)
- [ ] コード・手順を自分の環境でもう一度なぞったか(または「未検証」と明記)
- [ ] 変わりやすい情報に時点表記があるか

### 規約・法務

- [ ] 著者の経験・判断が背骨になっているか(Zenn「人が主体」原則、2026年3月方針)
- [ ] 収益に触れる場合、期間・コスト・前提が併記されているか
- [ ] 他者コンテンツの転載・過度な類似がないか(引用は出典明示)

### 品質

- [ ] TL;DRだけ読んでも価値があるか
- [ ] 「うまくいかなかったこと」の節があるか
- [ ] 声に出して読んで「自分の声」か(違和感のある段落はリライトへ戻す)

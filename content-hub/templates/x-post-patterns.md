# テンプレート: X投稿の型(パターン集)

生成時は [../prompts/x-single-post.md](../prompts/x-single-post.md) から型番号で参照する。
月次レビューで「伸びた型/滑った型」を判定し、この型集自体を更新していく(型は固定資産ではない)。

## 型一覧

### 型1: 実数Before/After

```text
(Before の苦痛を1行)

(やったこと1行)

結果:
・(実数1)
・(実数2)
・(実数3)

(ひとこと所感 or 問いかけ)
```

例の骨子: 「毎セッション同じ指示を10回 → hooks 47本で指示ゼロ」

### 型2: 失敗告白 → 学び

```text
正直に言う。(失敗の告白)

(何を間違えたか2〜3行)

今は(現在の状態)。

(同じ失敗をしそうな人への一言)
```

### 型3: あるある → 言語化

```text
(読者の「わかる」を突く日常の盲点1行)

(なぜそうなるかの言語化)

(自分はどう対処したか)

(共感の問いかけ)
```

### 型4: 損失回避(事実ベース限定)

```text
(知らないと損する事実・仕様1行)

(具体的に何が起きるか)

(回避方法)

(出典・時点)
```

注意: 事実で語れる場合のみ使用。煽り化しやすい型なので [../checklists/x-pre-post.md](../checklists/x-pre-post.md) 必須。

### 型5: 舞台裏・進捗ログ(個人開発向け)

```text
(今日・今週やったこと1行)

(過程のリアル: 詰まった点、意外だった点)

(数字があれば数字)

(次にやること)
```

### 型6: 持論の言い切り(対比構造)

```text
(世間の通念 or 過去の自分の考え)

でも(実際にやってわかったこと)。

(対比の言い切り。例「指示は消耗する。設計は蓄積する」)

(補足1〜2行)
```

### 型7: 図解・リスト保存型

```text
(テーマ): (要点の予告1行)

1. (要点)
2. (要点)
3. (要点)
4. (要点)
5. (要点)

保存して(具体的な使い所)で見返してほしい
```

画像(図解)を2〜4枚添付するとさらに有効(添付で評価向上・ブックマーク獲得)。

### 型8: 問いかけ・相談

```text
(自分の状況・迷いを自己開示)

(選択肢A) か (選択肢B) か。

(それぞれの自分なりの評価)

みんなはどうしてる?
```

リプライ誘発が最重要アルゴリズム要素のため、定期的に使う。

## 共通ルール

- 冒頭1行 = 記事タイトルの感覚で作り込む(折りたたみ対策)
- 外部リンクは本文に入れない(リプ欄へ)
- 絵文字2個まで、ハッシュタグ0〜1個
- 投稿時間帯の目安: 平日7〜9時 / 12時台 / 20〜22時(検証しながら調整)

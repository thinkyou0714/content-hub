# テンプレート: 有料note

生成時は [../prompts/note-paid-article.md](../prompts/note-paid-article.md) と併用する。
前提: テーマはZenn/Xで需要実証済みであること([../strategy/channels.md](../strategy/channels.md) のnote運用ルール)。

## 全体構造

```markdown
# タイトル(得られる結果 + 対象者 + 実数)

## ── 無料部分 ──

### 導入: 問題提起と共感
(読者の痛みの言語化。自分も同じ場所にいたことの自己開示)

### 結論の一部開示
(何をすれば解決するかの全体像。ここまでで1つ持ち帰れる価値を出す)

### 実績と前提の明示
(この方法でどうなったか。期間・コスト・環境・再現の前提条件を必ず併記)

### 有料部分の目次と対象者
(買うべき人・買わなくていい人を正直に書く)

## ── 有料部分 ──

### 完全な手順(ステップバイステップ)
(スクショ・設定値・コード込み。読者が迷わない粒度)

### テンプレート・成果物
(コピペで使えるプロンプト・設定ファイル・チェックリスト)

### 落とし穴と対処
(自分がハマった点。無料では書かなかった failure 込みの深掘り)

### 質問への対応方針
(コメント欄 or 他の窓口)
```

## 値付けの目安

- 売れ筋帯は100〜5,000円(市場調査より)。初出しは低めに置いて実績を作る
- 価格の最終決定は人間([../workflows/human-tasks.md](../workflows/human-tasks.md) HT-6)

## 公開前チェック(note特有)

- [ ] 無料部分単体で価値があるか(信頼の証明になっているか)
- [ ] 収益・実績表示に期間・コスト・前提があるか(景表法対応)
- [ ] 「買わなくていい人」を明示したか(返金・低評価の予防)
- [ ] Zenn無料記事との差分が明確か(二重売り防止)
- [ ] 機械的な量産に見えないか(note規約: 機械的大量投稿はアカウント停止対象)
